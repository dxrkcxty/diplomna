# Як запустити проєкт (для іншого комп’ютера)

## Що скинути

Папку **`kursova`** цілком (HTML, JS, CSS, `src`, `assets`).

Не обов’язково: `.idea`, `out`, `target` — їх можна не надсилати.

## Перед запуском

1. Встановити **Java 21** і **PostgreSQL**.
2. Створити БД (див. `DATABASE_SETUP.md`).
3. Скопіювати `src/main/resources/database.properties.example` → `database.properties` і вписати свій пароль БД.

## Запуск

1. У IntelliJ: **Run** → клас `Application`.
2. **Working directory** має бути папка `kursova` (де лежать `index.html` і `auth.js`).
3. Відкрити в браузері: **http://localhost:8080**

Якщо порт зайнятий — зупинити попередній запуск (Stop у IDE) або завершити процес `java.exe` на порту 8080.

## Категорії в БД

При старті сервера автоматично створюються 5 категорій у таблиці `categories`, і всім товарам проставляється `category_id` за полем `type`.

Перевірка в pgAdmin:

```sql
SELECT c.id, c.name, COUNT(p.id) AS products_count
FROM categories c
LEFT JOIN products p ON p.category_id = c.id
GROUP BY c.id, c.name
ORDER BY c.id;
```

## Посилання в проєкті

Усі сторінки, стилі та картинки — **лише файли всередині `kursova`** (`styles.css`, `assets/images/...`).  
Зовнішні соцмережі на головній (Telegram, Instagram) — за бажанням, на роботу магазину не впливають.

Якщо в БД уже були товари зі старими URL з інтернету, один раз виконай у PostgreSQL:

```sql
UPDATE products SET image_url = '/assets/images/product-default.svg' WHERE image_url LIKE 'http%';
```
